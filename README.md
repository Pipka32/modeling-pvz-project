# Проект по математическому моделированию работы ПВЗ

### [cite_start]Часть 1. Анализ предметной области [cite: 14]
* **Процесс**: Клиенты прибывают в пункт выдачи заказов. [cite_start]Если сотрудник свободен — заказ выдается сразу, если занят — клиент встает в очередь[cite: 15, 40].
* [cite_start]**Проблема**: Дефицит сотрудников приводит к длинным очередям и потере лояльности, а избыток — к лишним затратам на ФОТ при их простое[cite: 17, 18].
* [cite_start]**Цель**: Найти оптимальное количество сотрудников $N$, при котором среднее время ожидания минимально при разумных затратах[cite: 19].

### [cite_start]Часть 2. Математическая формализация [cite: 20]
* [cite_start]**Входные данные ($\lambda$)**: Интенсивность потока клиентов (чел/час)[cite: 24].
* [cite_start]**Среднее время обслуживания ($T_{cp}$)**: Среднее время на одного клиента[cite: 25].
* **Управляющий параметр ($N$)**: Количество работающих сотрудников[cite: 27].
* [cite_start]**Выходные метрики**: Среднее время ожидания ($Wq$) и загрузка персонала ($U$)[cite: 29, 30].

### [cite_start]Часть 3. Программная реализация (Симулятор) [cite: 33]
В коде реализованы:
* **Генератор событий**: Случайное время обслуживания через `random`[cite: 35, 36].
* **Диспетчер очереди**: Логика «Пришел клиент -> Есть свободный сотрудник? [cite_start]-> Начать выдачу : В очередь»[cite: 37, 40].
* [cite_start]**Счетчики**: Фиксация общего времени для расчета среднего ожидания[cite: 41, 43].

### [cite_start]Часть 4. Вычислительный эксперимент и решение [cite: 45, 46]
[cite_start]Запущена симуляция для разного количества сотрудников ($N = 1, 2, 3$)[cite: 47]:
* **При $N=1$**: Время ожидания превышает допустимые нормы[cite: 49].
* [cite_start]**При $N=2$**: Оптимальный баланс между скоростью и затратами[cite: 52].
* [cite_start]**При $N=3$**: Высокий простой сотрудников при незначительном выигрыше в скорости[cite: 49].

[cite_start]**Вывод**: Точка оптимизации — 2 сотрудника[cite: 52].
